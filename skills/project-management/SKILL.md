# 项目管理

## 目标

把项目维护成有状态、有完成标准、有下一步动作的推进单元。

## 原则

- 项目必须有完成标准；没有则标为模糊。
- 先判断继续、暂停、归档、拆分或合并。
- 下一步必须是一个可执行动作。
- 阻塞项必须说明缺口、责任或待决策点。
- 不主动扩大项目范围。
- 归档、合并、删除、改变目标必须先确认。

## 状态

- `进行中`：目标清楚，下一步明确
- `阻塞`：目标清楚，但缺少条件或决策
- `模糊`：目标、范围或完成标准不清
- `停滞`：长期无进展且原因不明
